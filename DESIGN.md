# Design System — ГРАФИН (QR-меню ресторана)

> Источник правды по дизайну. Любые визуальные/UI-решения сверять с этим файлом.
> Создано через /design-consultation (gstack), подтверждено на превью v4.

## Product Context
- **Что это:** цифровое меню ресторана, открывается по QR-коду на столике.
- **Для кого:** гость за столиком (моб. устройство) + владелец (редактирует контент через админку).
- **Пространство:** файн-дайнинг, премиальный сегмент.
- **Тип:** mobile-only веб-приложение (Next.js). Десктоп-вёрстка НЕ делается.
- **Ведущее ощущение:** «дорого и изысканно», тёплое и атмосферное; плотно, без лишнего воздуха.

## Платформа и охват
- **ТОЛЬКО мобильная вёрстка.** Одна колонка, крупные тач-зоны. Десктоп не проектируем.
- Контент русский — все шрифты обязаны иметь кириллицу.

## Aesthetic Direction
- **Направление:** тёплая «бумажная» роскошь + кинематографичный тёмный вход.
- **Уровень декора:** сдержанный (intentional) — тонкие хайрлайны, точки-выноски, мелкие капс-лейблы, латунь точечно.
- **Настроение:** приветствие — затемнённое видео зала с тёплым жёлтым светом (premium, cinematic); меню — тёплая бумажная карта с фото блюд.
- **Референсы:** Canlis (редакционная сдержанность), Atelier Crenn (тёмный + латунь).

## Typography
- **Display / Hero / заголовки категорий / названия блюд:** **Playfair Display** (высококонтрастный сериф, несёт «дорого»). Вес 600–700. Кириллица ✓.
- **Текст / описания / цены / лейблы / чипы:** **Manrope** (чистый гротеск). Вес 400–700. Кириллица ✓.
- **Загрузка:** Google Fonts (`Playfair+Display:ital,wght@0,500;0,600;0,700;0,800;1,500;1,600` + `Manrope:wght@400;500;600;700`).
- **Цены:** `font-variant-numeric: tabular-nums`.
- ⚠️ **Не использовать Instrument Sans и другие шрифты без кириллицы** — даёт фолбэк на системный (выглядит «стандартно»).
- Unbounded из системы исключён (тянул в «техно», конфликтовал с «дорого»).

### Шкала (mobile)
- Hero title: ~56px / line-height .95 (Playfair 700)
- Заголовок категории: ~22px (Playfair 700)
- Название блюда: ~17px (Playfair 600)
- Текст/описание: 13.5px (Manrope)
- Цена: 15px (Manrope 700, tabular)
- Лейблы/капс: 9–11px, letter-spacing .1–.3em, uppercase

## Color
- **Тёмный фон (приветствие):** `#0F0D0B` (тёплый почти-чёрный)
- **Бумага (фон меню):** `#EAE6DD` (тёплый off-white, НЕ белый, НЕ песочно-жёлтый)
- **Бумага-2 (плашки):** `#E3DECF`
- **Текст (ink):** `#211D17`
- **Приглушённый текст:** `#6E665A`
- **Латунь (акцент на бумаге):** `#A07C44`
- **Тёплое золото (акцент на тёмном):** `#D8B877`
- **Хайрлайн на бумаге:** `rgba(33,29,23,.14)`
- **Хайрлайн на тёмном:** `rgba(216,184,119,.26)`
- **Тёмная тема:** базовая схема и так тёмная на приветствии; меню остаётся светло-бумажным (печатная карта в руках).

## Layout
- **Подход:** mobile-only, одна колонка.
- **Приветствие:** фуллскрин — видео-фон зала + затемняющий градиент, контент прижат вниз (монограмма «Г» сверху, надзаголовок, крупное название, курсивный подзаголовок, блок о ресторане/шефе, золотая кнопка «Смотреть меню»).
- **Меню:** бумажный фон, липкая шапка с заголовком и **табами категорий** (горизонтальный скролл, активный — подчёркнут латунью). Категория = заголовок Playfair + хайрлайн + индекс. Блюдо = grid `1fr / 78px`: слева название с **точками-выноской до цены**, описание, бейджи + вес; справа фото 78×78 (radius 7px).
- **Скругления:** карточки/плашки 7–16px, кнопки 4px.
- **Плотность:** comfortable→compact. Зазоры между блюдами ~15px, без больших пустот.

## Badges (метки блюд)
- `Хит` — латунь (`rgba(160,124,68,.16)` / текст `#8a6a32`)
- `Остро` — терракот (`rgba(176,72,42,.13)` / текст `#a8502f`)
- `Веган` — зелёный (`rgba(110,140,66,.14)` / текст `#5b7a32`)
- Вес/граммовка — мелкий капс Manrope, `#8f8678`

## Motion
- **Подход:** intentional, сдержанно.
- Приветствие: мягкие fade / slide-up при появлении контента, плавный переход welcome → menu.
- Меню: минимум анимаций (скорость на 4G).
- Easing: enter `ease-out`, exit `ease-in`. Длительность: micro 80–120ms, short 180–250ms.
- **`prefers-reduced-motion`**: показывать постер вместо видео, отключать reveal-анимации.

## Видео-фон приветствия (спека для прода)
- Источник: `bg.MOV` → конвертирован. В прод (`public/`) кладём:
  - **`bg.mp4`** — H.264 (Main, yuv420p), без звука, `+faststart`, ~720×1280, ~2 МБ.
  - **`bg.webm`** — VP9, без звука, ~2.5 МБ.
  - **`poster.jpg`** — затемнённый первый кадр, ~110 КБ.
- Разметка:
  ```html
  <video autoplay muted loop playsinline preload="metadata" poster="/poster.jpg">
    <source src="/bg.webm" type="video/webm">
    <source src="/bg.mp4" type="video/mp4">
  </video>
  ```
- `muted` + `playsinline` обязательны (автоплей iOS). Порядок: webm → mp4. Поверх — затемняющий градиент.
- ❌ Не отдавать `.MOV` в прод (Chrome/Firefox не играют контейнер QuickTime).

## Performance
- Цель Lighthouse (mobile): Performance ≥ 90, Accessibility ≥ 90.
- Фото блюд через `next/image` (или оптимизированные), видео-фон ≤ 3 МБ, постер мгновенно.
- Приветствие открывается < 1.5 c на 4G.
